# 项目级 memory-management 初始化设计

## 背景

当前仓库已经采用 `docs/superpowers/specs/` 与 `docs/superpowers/plans/` 管理设计与实施文档，但尚未建立 `memory-management` 约定的项目内记忆结构。为了让后续问题解决经验、项目术语、关键上下文能够在仓库内持续沉淀，需要先初始化一套最小可用、低噪音的记忆骨架。

## 目标

- 在当前仓库根目录建立 `memory-management` 期望的标准文件与目录结构。
- 采用“空骨架”方式初始化，只提供必要标题、分区和简短说明，不写虚构示例内容。
- 让后续记录项目术语、关键人物、项目上下文和问题解决经验时有固定落点。
- 不影响现有代码、构建、测试和业务文档结构。

## 非目标

- 不在本次初始化阶段批量扫描仓库并自动生成大量项目记忆内容。
- 不创建全局共享知识库；本次仅初始化当前仓库内的项目级记忆结构。
- 不修改现有业务逻辑、配置或测试。
- 不预填任何可能失真的人物、术语或项目条目。

## 目录设计

在仓库根目录新增以下结构：

```text
CLAUDE.md
memory/
  glossary.md
  people/
  projects/
  context/
```

### CLAUDE.md

作为项目级“热缓存”，保留轻量、高频、便于快速解析的内容。初始化时包含以下区块：

- `# Memory`
- `## Me`
- `## People`
- `## Terms`
- `## Projects`
- `## Preferences`

每个区块只放用途说明或空表头，不写示例人物与示例术语。这样既符合 skill 的结构预期，也避免把演示内容误当成真实项目信息。

### memory/glossary.md

作为完整术语表与别名表，初始化时包含以下区块：

- `# Glossary`
- `## Acronyms`
- `## Internal Terms`
- `## Nicknames → Full Names`
- `## Project Codenames`

每个区块提供空表头与一行说明，便于后续持续追加。

### memory/people/

用于存放单个关键人物的详细资料文件。初始化阶段只创建目录，不创建示例人物文件。

### memory/projects/

用于存放单个项目、子项目或长期主题的详细上下文文件。初始化阶段只创建目录，不创建示例项目文件。

### memory/context/

用于存放团队、流程、工具约定等背景文档。初始化阶段只创建目录，不预建具体上下文文件，避免在没有确认信息时写入猜测内容。

## 内容风格

初始化内容遵循以下原则：

- **最小化**：只写结构，不写虚构案例。
- **可直接编辑**：首次人工补充时无需大改格式。
- **低噪音**：避免示例内容污染后续检索与理解。
- **兼容 skill 预期**：区块命名与用途尽量贴近 `memory-management` 的默认模型。

## 使用约定

初始化完成后，后续内容按以下方式沉淀：

- 高频术语、常用项目名、关键偏好优先写入 `CLAUDE.md`。
- 完整术语、缩写、别名写入 `memory/glossary.md`。
- 某个人需要更详细背景时，新增 `memory/people/<name>.md`。
- 某个项目或长期问题域需要详细背景时，新增 `memory/projects/<name>.md`。
- 团队流程、工具约定、环境说明等写入 `memory/context/` 下独立文件。

## 实施范围

本次实施仅包含：

- 新建 `CLAUDE.md`
- 新建 `memory/glossary.md`
- 新建 `memory/people/`
- 新建 `memory/projects/`
- 新建 `memory/context/`

不包含任何额外代码改动，也不包含自动填充仓库知识内容。

## 错误处理与兼容性

- 如果仓库后续已经存在同名文件，应采用非破坏性更新，保留已有真实内容。
- 本次由于当前仓库尚不存在这些路径，可直接创建。
- 新增文件均为纯 Markdown 或空目录，不会影响构建与运行时行为。

## 测试与验证

本次为文档与目录初始化，不需要业务测试；验证方式为：

- 仓库根目录可见 `CLAUDE.md`
- `memory/` 目录存在
- `memory/glossary.md` 存在且结构完整
- `memory/people/`、`memory/projects/`、`memory/context/` 目录存在
- 文件内容不包含虚构示例数据

## 验收标准

- 当前项目内已具备 `memory-management` 所需的最小骨架。
- 初始化结果干净、可读、可直接开始填充真实内容。
- 不引入与当前项目无关的示例、术语或人物信息。
- 不修改现有业务代码、测试或配置。
